⏺ What is weighted in-degree?                               
                                                                
  The matrix A is not a plain 0/1 adjacency matrix — each entry 
  is 1 / out_degree(src). So when you sum row i across all      
  columns:                                                      
                                                                
  in_deg[i] = sum of A[i, j] for all j                          
             = sum of (1 / out_degree(j)) for every node j that 
  links to i                                                    
                                                                
  This is "weighted" because each incoming link is not counted  
  as 1 — it is counted as a fraction based on how many other
  links the source node also has. A link from a node that only  
  links to you is worth more (1/1 = 1.0) than a link from a node
   that links to 1000 others (1/1000 = 0.001).

  This is actually very close to what PageRank itself computes —
   which is why the correlation is so high.
                                                                
  ---             
  Results interpretation (Spearman ρ = 0.865, Pearson r = 
  0.891):                                                       
         
  Both are strong but not perfect correlations. This tells three
   things:                                                      
   
  1. In-degree is a good proxy for PageRank — nodes with        
  more/higher-quality incoming links do tend to rank higher. The
   bulk of the scatter plot points follow a clear diagonal trend
   on the log-log axes.

  2. PageRank adds information beyond raw in-degree — the       
  correlation is not 1.0. Some nodes have high in-degree but
  lower-than-expected PageRank, because their inbound links come
   from low-authority sources. Conversely, some nodes with
  modest in-degree rank higher because they are linked to by
  highly authoritative nodes.

  3. Spearman < Pearson is unusual here — normally Spearman     
  (rank-based) is more robust. The fact that Pearson r (0.891)
  is slightly higher than Spearman ρ (0.865) suggests the       
  relationship is actually more linear in log-log space than it
  is purely ordinal, consistent with both quantities following
  power-law distributions.


  ⏺ Normally:    
  - Spearman ρ ≥ Pearson r, because Spearman is rank-based and more robust to outliers and
  non-linear relationships                                                                  
  - Pearson r can be dragged down by extreme outliers (like hub nodes with very high scores)
                                                                                            
  ---                                                                                       
  Here Pearson r (0.891) > Spearman ρ (0.865) — why?                                        
                                                                                            
  Both weighted in-degree and PageRank follow power-law distributions. When you plot them on
   log-log axes, the power-law relationship becomes a straight line:                        
                  
  log(PageRank) ≈ α × log(in_degree) + c                                                    
                                                                                            
  Pearson r measures linear correlation — and on log-log axes, the relationship IS linear.  
  So Pearson is actually well-suited here and gives a high score.                           
                                                                                            
  Spearman only looks at rank order — it ignores the actual values and just asks "does a    
  higher in-degree always mean a higher PageRank rank?". The hub nodes (extreme outliers)
  distort the rank ordering more than they distort the log-log linear fit, which slightly   
  pulls Spearman down.

  ---
  Simple analogy:
                                                                                            
  Imagine plotting city populations vs city areas on log-log axes — the relationship is a
  clean straight line. Pearson on the log-log values captures this well. But Spearman only  
  sees the rank order (city 1 is biggest, city 2 is second...) and loses the information
  that the relationship is precisely linear in log space.                                   
                  
  ---
  Bottom line: Pearson > Spearman here is actually evidence that the power-law structure is 
  real and clean — the log-log relationship is so linear that a linear correlation measure  
  outperforms a rank-based one.
                                                                                            
❯ What does both of them measures in the equation                                           
                                                                                        
⏺ Both Spearman and Pearson measure how strongly two variables move together, but in        
  different ways.                                                                           
                                               
  ---                                                                                       
  Pearson r — measures linear correlation on raw values:                                    
                                                                                            
  r = Σ[(x_i - x̄)(y_i - ȳ)] / sqrt[Σ(x_i - x̄)² × Σ(y_i - ȳ)²]                               
                                                                                            
  - x = weighted in-degree values                                                           
  - y = PageRank scores                                                                     
  - Asks: "when in-degree increases by some amount, does PageRank increase proportionally?" 
  - Sensitive to the actual magnitude of values                                             
  - Range: -1 to 1                                                                          
                                                                                            
  ---                                                                                       
  Spearman ρ — measures rank correlation:

  ρ = 1 - (6 × Σd_i²) / (n(n²-1))
                                                                                            
  - d_i = difference in rank between node i's in-degree rank and PageRank rank              
  - Asks: "when in-degree rank goes up by one position, does PageRank rank also go up?"     
  - Ignores actual values, only cares about ordering                                        
  - Range: -1 to 1                                                                          
                                                                                            
  ---                                                                                       
  Key difference in this context:
                                                                                            
  ┌─────────────────────────┬──────────────────────────┬───────────────────────────────┐ 
  │                         │         Pearson          │           Spearman            │    
  ├─────────────────────────┼──────────────────────────┼───────────────────────────────┤ 
  │ Input                   │ Actual log(score) values │ Rank positions (1st, 2nd,     │ 
  │                         │                          │ 3rd...)                       │ 
  ├─────────────────────────┼──────────────────────────┼───────────────────────────────┤    
  │ Question                │ Is the relationship      │ Is the ordering consistent?   │    
  │                         │ linear?                  │                               │    
  ├─────────────────────────┼──────────────────────────┼───────────────────────────────┤    
  │ Affected by outliers    │ Yes                      │ No                            │ 
  ├─────────────────────────┼──────────────────────────┼───────────────────────────────┤ 
  │ Affected by             │ Yes                      │ No                            │ 
  │ non-linearity           │                          │                               │
  └─────────────────────────┴──────────────────────────┴───────────────────────────────┘    
   
  Pearson on log-log values says: "the log values move together linearly"                   
                  
  Spearman says: "the rank positions move together consistently"                            
                  
  Here both are high (0.891 and 0.865), meaning weighted in-degree and PageRank agree both  
  in magnitude and ordering — just slightly more in magnitude than in pure rank order.
                                                                                            
❯ should spearman be higher than pearson then                                               
                                                                                        
⏺ Yes, normally Spearman ≥ Pearson because:                                                 
                                                                                            
  - Pearson assumes a linear relationship and is sensitive to outliers pulling it down      
  - Spearman only cares about rank order, so extreme hub nodes (very high in-degree, very   
  high PageRank) don't distort it — they just become rank 1, rank 2, etc.                   
                                                                                            
  ---                                                                                       
  Why it flipped here:                                                                      
                                                                                            
  The hub nodes (extreme outliers) actually help Pearson rather than hurt it. On log-log
  axes, those extreme points fall perfectly on the straight line — they are not outliers in 
  log space, they are the most consistent part of the relationship.
                                                                                            
  But for Spearman, the hub nodes cause rank ties and instability at the top of the ranking.
   A node might be rank 1 by in-degree but rank 3 by PageRank (or vice versa), because at
  the extreme end small differences in the recursive authority calculation reorder the top  
  nodes. These rank swaps penalise Spearman even though the actual score values are very
  close.

  ---
  So the flip tells you:
                                                                                            
  ▎ The relationship between weighted in-degree and PageRank is more consistent in log-scale
   magnitude than it is in exact rank order                                                 
                  
  Meaning the scores track each other well proportionally across the whole graph, but the   
  precise ordering of the top nodes is less stable — which makes sense since top nodes have
  very similar PageRank scores and small differences in recursive authority can shuffle     
  their exact positions.

  ⏺ The analytical solution comes from the same dataset as the iterative solution —
  data/web-Google.txt (or the toy graph in --small mode). Both methods run on identical 
  input.                                                                                     
                                               
  However there is an important detail at evaluate.py:126-142:                               
                                                                                             
  analytical = AnalyticalPageRank(A, dangling, p=p_ref)                                      
  r_analytical = analytical.compute()                                                        
  if r_analytical is not None:                                                               
      # use analytical as ground truth                                                       
  else:                                                                                      
      # graph too large for analytical — fall back to p=0.10 as proxy
      r_b = results_by_p[0.10]                                                               
                                                                                             
  The analytical solution uses a direct linear solver (scipy.sparse.linalg.spsolve) to solve:
                                                                                             
  r = (1-p) * A * r + p * (1/N)                                                              
                                                                                             
  This only works if the matrix fits in memory and the solver can handle it. For the full    
  web-Google graph (875k nodes), r_analytical returns None if the solver fails — in which    
  case the p=0.10 iterative run is used as a proxy ground truth instead.                     
                  
  So the ranking metrics (Spearman ρ = 0.993, Kendall τ = 0.941) are actually comparing      
  p=0.15 vs p=0.10 iterative results, not iterative vs true analytical — unless the
  analytical solver succeeded on your machine.                                               
                  

It means the color value for each point is not raw r, but transformed:

ci=log(1+ri)

Why:

PageRank values are usually tiny and highly skewed.
If you color by raw r, most points look nearly same color.
log(1+r) spreads small values more evenly for visual contrast.
So color still reflects PageRank magnitude, just on a compressed/log scale.
log1p(x) is just a numerically stable way to compute log(1+x).
