# setdefault on _rev_adj[2] creates new [] for radium, then appends 0 → [0]

print(f"_rev_adj: {kg._rev_adj}")  # {1: [0], 2: [0]}  # Both polonium and radium point back to Marie Curie

# Third triple: "radium" -> "used_in" -> "radiotherapy"
# s_entity.idx = 2 (radium), o_entity.idx = 3 (radiotherapy)

# setdefault on _adj[2] creates new [] for radium, then appends 3 → [3]
# setdefault on _rev_adj[3] creates new [] for radiotherapy, then appends 2 → [2]

print("\nAfter third triple:")

Below is a detailed explanation of the adjacency_matrix method in the KnowledgeGraph class. This method generates a sparse adjacency matrix suitable for PageRank and other graph algorithms. I'll break it down step by step.

Method Signature and Purpose
Purpose: Returns a column-stochastic adjacency matrix (N x N) where N is the number of entities.
Column-stochastic: Each column sums to 1 (probabilities for random walks).
CSR format: Compressed Sparse Row - efficient for sparse matrices.
weighted parameter: If True, uses edge weights; if False, treats all edges as weight 1.

Step 1: Caching Check
Checks if the matrix is already computed and up-to-date.
self._dirty is set to True whenever the graph changes (e.g., in add_triple).
If clean and cached, returns the existing matrix for performance.

Step 2: Handle Empty Graph
If no entities, returns an empty (0x0) sparse matrix.

Step 3: Count Out-Degrees
Creates an array to store out-degree (number of outgoing edges) for each entity.
For each relation, increments the source entity's out-degree by the edge weight (or 1 if unweighted).
Out-degree: How many edges leave each node.

Step 4: Build Matrix Data
Prepares data for the sparse matrix in COO (Coordinate) format.
For each relation:
src: source entity index (column in matrix)
dst: destination entity index (row in matrix)
w: edge weight (or 1)
Only processes if source has outgoing edges (out_degree[src] > 0)
Matrix entry: A[dst, src] = w / out_degree[src]
This is the probability of following this edge from src to dst
Normalized by out-degree so each column sums to 1

Step 5: Create Sparse Matrix
Converts the COO data to CSR format.
shape=(N, N): Square matrix with N entities.
Matrix interpretation:
A[i, j] = probability of going from entity j to entity i
For PageRank: represents the transition matrix

Step 6: Handle Dangling Nodes
Dangling nodes: Entities with no outgoing edges (out_degree == 0).
In PageRank, these nodes distribute probability equally to all nodes.
For each dangling node d:
Adds 1/N to every entry in column d (i.e., A[i, d] += 1/N for all i).
This ensures each column still sums to 1.

Step 7: Finalize and Cache
Converts to CSR format (if not already).
Caches the matrix and marks as clean.
Returns the adjacency matrix.

Example Usage
This matrix can be used directly in PageRank algorithms or other graph computations. The column-stochastic property ensures it's a valid transition matrix for random walks.


What is data, data.append(w / out_degree[src])

data

is a list that stores the matrix values in COO (Coordinate) format for building the sparse adjacency matrix.

In the line data.append(w / out_degree[src]):

w is the edge weight (relation weight if weighted=True, otherwise 1.0)
out_degree[src] is the number of outgoing edges from the source entity
w / out_degree[src] normalizes the weight so that all outgoing edges from a source sum to 1 (column-stochastic property)
This creates the transition probabilities for PageRank: each entry represents the probability of following that specific edge from the source node. The data list, along with rows (destination indices) and cols (source indices), is used to construct the sparse matrix A where A[dst, src] = w / out_degree[src].


The line A = sp.csr_matrix((data, (rows, cols)), shape=(N, N), dtype=np.float64) creates a sparse adjacency matrix in CSR (Compressed Sparse Row) format using SciPy.

data: List of matrix values (normalized weights w / out_degree[src])
rows: List of row indices (destination entity indices)
cols: List of column indices (source entity indices)
shape=(N, N): Matrix dimensions (N x N where N = number of entities)
dtype=np.float64: Data type for the matrix values (64-bit float)
This constructs the transition matrix where A[i, j] represents the probability of moving from entity j to entity i. The CSR format is efficient for sparse matrices and supports fast matrix-vector multiplications needed for PageRank iterations. The input (data, (rows, cols)) is in COO (Coordinate) format, which SciPy automatically converts to CSR.




The sp.csr_matrix((data, (rows, cols)), shape=(N, N), dtype=np.float64) call creates a sparse matrix using SciPy's COO (Coordinate) format input, which is then internally converted to CSR (Compressed Sparse Row) format for efficiency.

How It Works
The three arrays work together to specify non-zero matrix entries:

data: 1D array of values (e.g., [0.5, 0.5, 1.0])
rows: 1D array of row indices (e.g., [1, 2, 3])
cols: 1D array of column indices (e.g., [0, 0, 2])
Each triplet (data[i], rows[i], cols[i]) places the value at that matrix position.

Example
Suppose we have:

data = [0.5, 0.5, 1.0]
rows = [1, 2, 3]
cols = [0, 0, 2]
shape = (4, 4) (4x4 matrix)
This creates:
[[0.  0.  0.  0.]   # Row 0
 [0.5 0.  0.  0.]   # Row 1: data[0]=0.5 at (1,0)
 [0.5 0.  0.  0.]   # Row 2: data[1]=0.5 at (2,0)  
 [0.  0.  1.  0.]]  # Row 3: data[2]=1.0 at (3,2)

In Your Code
In the adjacency_matrix method:

rows contains destination indices (dst)
cols contains source indices (src)
data contains normalized weights (w / out_degree[src])
Each entry A[dst, src] = w / out_degree[src] represents the transition probability from source to destination
The matrix is column-stochastic (each column sums to 1) for PageRank random walks. Only non-zero entries are stored, making it memory-efficient for sparse graphs.



The code handles "dangling nodes" in the PageRank adjacency matrix. Dangling nodes are entities with no outgoing edges (out_degree == 0). In PageRank, these nodes need to distribute their probability uniformly to all nodes to maintain the random walk.

Step-by-Step Explanation
Find dangling nodes:

np.where(out_degree == 0) returns indices where out_degree is 0
[0] extracts the array of indices
dangling is an array of entity indices with no outgoing edges
Check if any exist:

Only proceed if there are dangling nodes
Create row indices:

np.arange(N) creates [0, 1, 2, ..., N-1]
np.tile(array, len(dangling)) repeats this array for each dangling node
For N=3, dangling=[1]: dan_rows = [0, 1, 2] (one set for dangling node 1)
Create column indices:

np.repeat(dangling, N) repeats each dangling index N times
For dangling=[1], N=3: dan_cols = [1, 1, 1] (column 1 repeated 3 times)
Create data values:

np.full(size, value) creates array filled with 1.0/N
For N=3: dan_data = [0.333, 0.333, 0.333]
Create and add sparse matrix:

Creates a sparse matrix with 1/N in every row of each dangling node's column
Adds it to the existing matrix A
Example for N=3, dangling=[1]
dan_rows = [0, 1, 2]
dan_cols = [1, 1, 1]
dan_data = [0.333, 0.333, 0.333]
This adds a column of 1/3 values to column 1 of matrix A, meaning a random surfer at the dangling node jumps to any node with equal probability. This ensures each column sums to 1 for proper PageRank convergence




np.tile and np.repeat are used to create the coordinate arrays for a sparse matrix that adds a uniform distribution (1/N in each row) to the columns of dangling nodes

Why np.tile(np.arange(N), len(dangling)) for dan_rows:
np.arange(N) creates [0, 1, 2, ..., N-1] (all row indices)
np.tile(array, len(dangling)) repeats this sequence once for each dangling node
Result: For each dangling node, we get all row indices [0,1,2,...,N-1]
Purpose: We need to add 1/N to every row in the dangling node's column

Why np.repeat(dangling, N) for dan_cols:
dangling is an array of dangling node indices, e.g., [1, 3]
np.repeat(array, N) repeats each element in the array N times
Result: For each dangling node, its index is repeated N times
Purpose: Each dangling node's column gets N entries (one per row)

Example for N=3, dangling=[1]:
dan_rows = np.tile([0,1,2], 1) = [0, 1, 2] (all rows for dangling node 1)
dan_cols = np.repeat([1], 3) = [1, 1, 1] (column 1 repeated for each row)
dan_data = [0.333, 0.333, 0.333] (1/3 in each position)
This creates a sparse matrix that adds 1/3 to positions (0,1), (1,1), and (2,1), giving a uniform distribution column for the dangling node. The same pattern repeats for multiple dangling nodes.
