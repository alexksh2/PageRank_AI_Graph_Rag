The file implements a GraphRAG Query Engine that uses Personalized PageRank (PPR) for multi-hop query answering over a knowledge graph (KG). It extracts seed entities from queries, runs PPR to rank relevant entities, reconstructs reasoning chains, and generates explanations—enabling retrieval-augmented generation (RAG) without external APIs.

Block 7: _default_seed_extractor Method (Lines 151-166)
Purpose: Default implementation for extracting seed entities from a query string via keyword matching.

Key Details:

Matching: Case-insensitive substring search against all KG entity names (e.g., "Marie Curie" in "Who discovered radium?").
Sorting: Prioritizes longer matches (e.g., "Marie Curie" over "Marie" if both match).
Capping: Limits to 5 seeds to avoid over-biasing.
Limitations: Simple; production would use NER/LLMs for better accuracy.
Block 8: _build_chains Method (Lines 168-183)
Purpose: Reconstructs multi-hop reasoning chains (paths) from seeds to top retrieved entities.

Key Details:

Targets: Filters top entities to exclude seeds (focus on new discoveries).
Path Finding: For each seed-target pair (limited to 5 targets/seed), uses explain_path (DFS-based) to find all simple paths up to max_hops.
Capping: Limits to 10 total chains to avoid explosion.
Output: List of paths (e.g., [["seed", "intermediate", "target"]]) for explainability.

Key Details:

Components: Extracts top 3 entity names, formats seeds/chains.
Content: Explains PPR propagation, ranking criteria, and includes the shortest chain if available.
Output: A readable string tying back to the query and KG structure.
